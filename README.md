# 🧪 Proyecto QA: Testing Manual y Automatizado - App de Viajes

Este proyecto consiste en la validación manual y automatizada de una aplicación web de viajes desarrollada en HTML, CSS y JavaScript. 
El objetivo es asegurar la calidad del producto, detectar errores funcionales y mejorar la experiencia del usuario.

## 🌐 Demo de la app
🔗 [Ver aplicación](https://nathy25.github.io/Viajes)

## 🎯 Objetivos del proyecto
- Aplicar técnicas de testing manual sobre una app real.
- Documentar casos de prueba y errores encontrados.
- Explorar herramientas de automatización para pruebas básicas.
- Demostrar habilidades en QA para roles profesionales.

## 🧰 Herramientas utilizadas
- HTML, CSS, JavaScript
- Git y GitHub
- Jest (para pruebas unitarias)
- Cypress (opcional, para pruebas end-to-end)
- Markdown (para documentación de pruebas)

## 📝 Casos de prueba manual
| ID | Funcionalidad | Resultado esperado | Resultado real | Estado |
|----|----------------|--------------------|----------------|--------|
| TC01 | Navegación entre secciones | Redirige correctamente | OK | ✅ |
| TC02 | Formulario de búsqueda | Muestra resultados válidos | Error en validación | ❌ |
| TC03 | Enlaces externos | Abren en nueva pestaña | OK | ✅ |
| TC04 | Visualización de imágenes | Se cargan correctamente | OK | ✅ |

> 📌 Los errores fueron documentados con capturas de pantalla en la carpeta `/evidencias`.

## ⚙️ Automatización 
- Pruebas unitarias con Jest para funciones de búsqueda.
- Pruebas end-to-end con Cypress:
  - Carga de la página principal
  - Interacción con el formulario
  - Validación de resultados
